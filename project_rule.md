# GitHub工具合集 - 待办事项

## 项目设计理念

**极简原则：** 每个工具独立简单，1个代码文件，且不超过120行。如果超过，则拆分项目。

**调用规则：** 子项目间禁止互相调用，只有根目录的代码文件有这个权限。

**风格规则：** 代码风格跟着github-checker走，允许checker允许的，要求checker要求的。

**适用范围规则：** 子项目必须遵守上述所有规则。根目录的独立代码文件（如count_lines.py、run.py等）和trace目录的代码文件不受这些规则限制（不限制代码文件个数、行数、风格、调用限制等）。

## 核心原则

### 1. 极简原则

- 每个工具1个代码文件
- 单文件不超过120行
- 超过则拆分项目

### 2. 独立原则

- 子项目间禁止互相调用
- 子项目不能引用其他子项目的模块

### 3. 一致性原则

- 代码风格跟着github-checker走
- github-checker允许的风格，其他项目都允许
- github-checker要求的规范，其他项目都需遵守

### 4. 隔离原则

- github-checker-检测状态 保持原样，不做任何修改
- 其他子项目可以自由修改、重构

## 调用规则

### 禁止事项

- 子项目之间禁止互相调用
- 子项目不能导入其他子项目的模块
- 子项目不能引用其他子项目的路径

### 允许事项

- 根目录工具可以调用任意子项目
- 根目录工具可以访问所有子项目的代码文件
- 根目录工具可以收集所有子项目的运行状态

### github-checker特殊规则

- github-checker-检测状态 保持原样，不修改任何内容
- 但可以在根目录调用 github-checker-检测状态
- GitHub-checker-style-代码检测 是独立的代码风格检测子项目
